# Orbit-simulation

`Orbit-simulation` — программный модуль для численного моделирования движения искусственного спутника Земли на солнечно-синхронной орбите.

---

Проект моделирует движение спутника по низкой околоземельной орбите с учётом:

- центрального ньютоновского гравитационного взаимодействия;
- релятивистских поправок первого постньютоновского порядка;
- возмущения от второй зональной гармоники гравитационного поля Земли \(J_2\).

Основное прикладное назначение модели — расчёт и проверка параметров солнечно-синхронной орбиты.

---

## Расчёт солнечно-синхронной орбиты

Солнечно-синхронная орбита подбирается таким образом, чтобы скорость прецессии долготы восходящего узла \(\dot\Omega\), обусловленная \(J_2\)-возмущением, совпадала со средней угловой скоростью движения Земли вокруг Солнца:

\[
\dot\Omega(i) = n_{\odot}.
\]

Для расчёта используется выражение для вековой прецессии RAAN:

\[
\dot\Omega
=
-\frac{3}{2}
J_2
\left(\frac{R}{p}\right)^2
n
\cos i,
\]

где

\[
p=a(1-e^2),
\qquad
n=\sqrt{\frac{\mu}{a^3}}.
\]

Здесь \(a\) — большая полуось, \(e\) — эксцентриситет, \(i\) — наклонение орбиты.

Целевое значение средней угловой скорости Солнца принимается равным

\[
n_{\odot}
\approx
0.985647^\circ / \text{сут}.
\]

Наклонение орбиты определяется численно методом бисекции из уравнения

\[
\dot\Omega(i) - n_{\odot}=0.
\]

Для орбиты с высотой 800 км и эксцентриситетом \(e=0.001\) расчётное наклонение составляет приблизительно

\[
i \approx 98.6^\circ.
\]

## Численное интегрирование

Для интегрирования уравнений движения используется явный [метод](https://github.com/zachetienne/nrpytutorial/blob/master/Tutorial-RK_Butcher_Table_Dictionary.ipynb) Рунге–Кутты восьмого порядка с 13 стадиями.

Шаг интегрирования постоянный и равен 60 секундам.

---

## Начальные условия

Начальные положения и скорости Земли заданы по эфемеридам NASA/JPL Horizons на эпоху

\[
\text{2026-Apr-30 00:00:00.0000 TDB},
\]

что соответствует

\[
JD = 2461160.5.
\]

## Выходные данные

В процессе моделирования вычисляются и сохраняются орбитальные элементы спутника.

Выходной файл:

```text
orbit_output.txt
```

Формат строки:

```text
t_days  jd_tdb  distance_km  speed_km_s  a_km  e  i_deg  Omega_deg  omega_deg  nu_deg
```

Описание полей:

| Поле | Описание |
|---|---|
| `t_days` | время от начала моделирования, сут |
| `jd_tdb` | юлианская дата в шкале TDB |
| `distance_km` | расстояние от спутника до Земли, км |
| `speed_km_s` | относительная скорость спутника, км/с |
| `a_km` | большая полуось, км |
| `e` | эксцентриситет |
| `i_deg` | наклонение, град |
| `Omega_deg` | долгота восходящего узла, град |
| `omega_deg` | аргумент перицентра, град |
| `nu_deg` | истинная аномалия, град |

## Сборка

Требования:

| Компонент | Версия |
|---|---:|
| C++ | C++17 |
| CMake | 3.14 или выше |

Команды сборки:

```bash
cmake -S . -B build
cmake --build build
```

---

## Запуск

```bash
./build/orbit_simulation
```

Программа записывает результаты интегрирования в файл `orbit_output.txt`.

Для построения графиков используется скрипт:

```bash
python scripts/plot_orbit.py
```

Скрипт строит графики:

- узловой прецессии;
- расстояния от спутника до Земли;
- LTAN-drift.
